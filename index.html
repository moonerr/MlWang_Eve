The 'Actual'

How does the suround sound work?

Hardware


  1. Force Sensitive Resistors (FSR) * 2

    2. FSR is Used for detecting the pressure applied by two hands. It converts the force to an analog voltage. Arduino converts the analog voltage into digital one, ranged from 0 to 1023. The harder users squeeze FSR, the bigger the number is.

  2. Arduino Uno

    3. It is a basic set for makers, powered by a USB porter or any 5V power port. It is used as a controler to cope with inputs and outputs. It receives the signals from FSR and sends them to laptop(Processing).

  3. Arduino Lilypad

    4. Lilypad gives back its acceleratio in Y-axis when it rolls. And it will send to processing as a input to control the balance of music.

  4. Laptop (processing) with speakers

    5. Lap top is used for executing Processing code and give out sound with its two-channel speakers.

# Code with Arduino

  1. Read the value from FSR

    2. Connect the FSR to Arduino with A0, A1 as input port. Get the value from the two pins by method analogRead(PIN). 

                            
                        int fsrAnalogPinLeft = A0; // Left FSR is connected to analog 0
                        int fsrAnalogPinRight = A1; // Right FSR is connected to anolog 1
                        int fsrReadingLeft, fsrReadingRight;      // the analog reading from the FSR resistor divider
            
                        void setup() {
                        // Init USB serial port for debugging
                        Serial.begin(9600);
                        }
            
                        void loop() {
                        //get the fst number
                        fsrReadingLeft = analogRead(fsrAnalogPinLeft);
                        fsrReadingRight = analogRead(fsrAnalogPinRight);
                        }
  2. Prepare the value for communication with Processing

    3. The virable Adjust is created to calculate the difference between the values of pressure for each side. Thus, the first step is to use one value to minus the other.
    Then because the default frame rate of Processing is 60, which is to say, it will read about every 20ms. So secondly, it is essential to set a timmer for the arduino to send the value every 50ms, which should be no less than 20ms.
    It is also important to check the writing capability of Arduino Uno. While using Serial.write(), the data is sent by One byte each time. So it is why in here Adjust was mapped to 0 ~ 28-1. 

                            
                        //set a timer
                        unsigned long timer = 0;
                        void loop() {
                            ...
                            
                            //prepare the value of adjuat
                            int Adjust;
                            Adjust = (fsrReadingRight - fsrReadingLeft);//The stronger side wins
                            Adjust = map(Adjust, -1000, 1000,0,255); //map to fit the O/I limitation
                            
                            //Let the arduino send the value every 50ms
                            if((millis()-timer)>50){ 
                                if (fsrReadingRight != 0 || fsrReadingLeft != 0){
                                    Serial.write(Adjust);}
                                timer = millis();  
                            }
                        }

# Code with Processing

  1. Import libraries

    2. The serial library is used for communicating with Arduino. The sound library is used for control the music. 

                            
                        import processing.serial.*;
                        import processing.sound.*;
  2. Prepare variables and setup the port

    3. There are much more balls connected, so the port in use are more than two. myportN is used to contain the port from the port list. posN is used to store the value sent by Arduino. It is surprising to find that when we use USB hubs, the port name will expand by two numbers. So it will be better to use port list to get their names.
    SoundFile() is also used in setup block to bind the virable to the music file. It is also worth of pointing out that the sound file should be mono sound. Stereo sound would not work. 

                            
                        Serial myPort01, myPort02, myPort03, myPort04, myPort05;// Create object from Serial class
                        int pos01, pos02, pos03, pos04, pos05;// Data received from the serial port
                        SoundFile player01, player02;
                        
                        void setup () {
                            
                            size(512, 200);
                            String portName01 = Serial.list()[4]; // arduino uno 142101
                            String portName02 = Serial.list()[5]; // lilypad 1 1422101
                            //String portName03 = Serial.list()[6]; // lilypad 2 1422201
                            //String portName04 = Serial.list()[7]; // lilypad 1 1422101
                            //String portName05 = Serial.list()[8]; // lilypad 2 1422201
                            
                            //String portName = "/dev/cu.usbmodem1422101"; // check the portname of arduino
                            myPort01 = new Serial (this, portName01, 9600); // set the com port and the baud rate according to the Arduino IDE
                            myPort02 = new Serial (this, portName02, 9600);
                            //myPort04 = new Serial (this, portName04, 9600);
                        
                            player01 = new SoundFile(this, "byhmono.wav"); // set sound for squeeze
                            player02 = new SoundFile(this, "soundmono.wav");// set sound for swing
                        
                            // frameRate(30); initial value is 20 fps 
                        }
  3. Play the music with balance

    4. port.available() is used to check whether the port is receiving data from Arduino. When it is working, then it is the time to read and store it into posN. When it gets the data, the music file can be played if it is not turned on. This piece is to avoid lag music.
    Then next, use map(variable, v1, v2, va, vb) to release the value of Adjust. For the range of music balance is from -1 to +1, Adjust is mapped to suit into this range.
    The last step, use playerN.pan(AdjustN) to control the music balance! 

                            
                    void draw () {
                        if ( myPort01.available() > 0) {  // If data is available,
                            pos01 = myPort01.read();  // read it and store it in pos
                            if(!player01.isPlaying()) { 
                                player01.play();}
                            float adjust01 = map( float(pos01), 0, 255, -1.0, 1.0);
                            println("adjust01"); println(adjust01);
                            player01.pan(adjust01);}
                            else if ( myPort01.available() <= 0 && player02.isPlaying()){
                            player01.pause(); }
